Web Accessibility Checklist
User Characteristic Accessible Design Tip
Unable to See Code all images with ALT text
Unable to perceive color Avoid using color alone to convey information
Unable to use mouse Be sure all website features can be accessed using keyboard
Unable to hear Add captions to multimedia
Prone to having seizures Avoid content that flashes
Easily distractable Keep the design simple

Accessibility Review of the Nixa High School Home Page

  • ●[Severe] Controls and media elements should have labels (1)
  • ●[Warning] Avoid positive integer values for tabIndex (4)
  • ●[Warning] Images should have an alt attribute (6)
  • ●[Warning] Meaningful images should not be used in element backgrounds (4)
  • ●[Warning] Text elements should have a reasonable contrast ratio (68)
  • ●[Warning] The purpose of each link should be clear from the link text (8)
  • ●[Warning] The web page should have a title that describes topic or purpose (1)
  • ●[Warning] Some elements are focusable but either invisible or obscured by another element (2)
  • ﻿
  • ●[Passing] ARIA state and property values must be valid
  • ●[Passing] Elements with ARIA roles must use a valid, non-abstract ARIA role
  • ●[Passing] ARIA attributes which refer to other elements by ID should refer to elements which exist in the DOM
  • ●[Passing] Elements with ARIA roles must have all required attributes for that role
